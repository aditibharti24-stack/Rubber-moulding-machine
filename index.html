Implementation Flow

How We Implement SmartMold

1. Instrument The Machine Install temperature, pressure, and vibration sensing points on the molding press.
2. Collect Through PLC / ESP32 Read field values locally and package the telemetry as MQTT messages.
3. Transmit To Server Storage Push readings through the Industrial LAN to the broker, server, and database layer.
4. Analyze And Act Convert stored history into dashboards, alerts, and statistical maintenance insight.

Why The Upgrade Matters

Traditional rubber molding machines run with fixed cycles and low visibility, leading to inconsistent quality, delayed fault detection, and reactive maintenance.

What Makes This Different

The SmartMold stack connects sensing, secure networking, relational storage, and statistics into one workflow so the machine can be monitored and improved using evidence.

PLC / ESP32 / Arduino — MQTT Client

Collects field readings, packages payloads, buffers data locally when disconnected, and reconnects to publish queued packets to the broker.

Client–Server Data Flow

Sensors → PLC (MQTT Client) → Industrial LAN → MQTT Broker → Central Server → PostgreSQL DB → Dashboard

Sensors→PLC→LAN→Broker→Server→PostgreSQL→Dashboard

Client Side

  • Publishes to topics: /machine/temperature, /machine/pressure, /machine/vibration
  • Uses QoS Level 2 for exactly-once delivery
  • Buffers locally when disconnected and flushes on reconnect
  • Supports username-password authenticated broker access

Server Side

  • MQTT broker subscribes to /machine/# wildcard topics
  • Central server validates incoming values and timestamps
  • PostgreSQL stores the normalized relational record set
  • Dashboard consumes the live stream for monitoring and alerts

Why MQTT

MQTT is lightweight, event-oriented, and better suited to continuous industrial telemetry than HTTP, raw TCP payloads, or unreliable datagrams.

Fault Tolerance

Local buffering, reconnect logic, acknowledgments, and QoS 2 reduce packet loss during network interruptions.

Security (IEC 62443)

TLS 1.3 encrypted transport, authenticated broker access, and secure topic flow support industrial cybersecurity good practice.

-- Auto-create critical temperature alerts
CREATE TRIGGER auto_alert_on_threshold
AFTER INSERT ON SensorReading
FOR EACH ROW
BEGIN
  IF NEW.value > 200 THEN
    INSERT INTO Alert(sensor_id, triggered_at, alert_type, threshold_value, actual_value, resolved)
    VALUES (NEW.sensor_id, NOW(), 'CRITICAL_TEMP', 200, NEW.value, FALSE);
  END IF;
END;
